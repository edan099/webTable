# 📤 Chrome Web Store 提交指南

## 🎯 快速开始

### 第 1 步：修改 manifest.json

⚠️ **重要**：在提交前，请修改以下两个字段：

```json
"author": "Your Name",          ← 改为您的名字
"homepage_url": "https://...",  ← 改为您的 GitHub 或网站（可选）
```

**如果没有 GitHub 或网站，可以删除 `homepage_url` 这一行。**

---

### 第 2 步：准备权限说明文本

**位置：** Chrome Web Store 开发者控制台 → 您的扩展 → 隐私权规范标签页

**复制以下文本：**（已保存在 `WEBSTORE_PERMISSION_TEXT.txt`）

```
本扩展程序需要访问所有网站以实现其核心功能：识别和提取网页表格数据。

【权限必要性】
用户可能在任何网站上遇到需要提取的表格，包括但不限于：企业管理系统、数据分析平台、在线文档、电商网站等。无法预先限定特定域名，因为表格可能出现在互联网的任何位置。

【实际用途】
1. 检测页面中的 HTML 表格和组件库表格（Element UI、Ant Design）
2. 在表格上方显示提取按钮（非侵入式）
3. 仅在用户主动点击时读取表格内容
4. 将数据转换为 JSON、CSV、SQL 格式

【隐私保护】
• 完全本地运行，不上传任何数据
• 不收集用户浏览历史
• 不访问密码或敏感信息
• 只读取用户明确要求提取的表格
• 开源代码，可供审计

【用户控制】
用户完全控制何时使用提取功能，仅在点击提取按钮时才读取数据，可随时禁用扩展程序。
```

```bash

```

**方法 2：手动压缩**
```bash
zip -r webTable-v1.2.0.zip . \
  -i "manifest.json" \
  -i "content.js" \
  -i "style.css" \
  -i "icon*.png" \
  -i "README.md" \
  -i "LICENSE"
```

⚠️ **重要**：压缩包内必须直接包含 `manifest.json`，不能嵌套在文件夹中！

## 📋 提交流程

### 1. 登录开发者控制台

访问：https://chrome.google.com/webstore/devconsole

### 2. 创建新项目

- 点击「新增项」
- 上传 ZIP 文件

### 3. 填写商品详情

- 商品信息（名称、描述）
- 图形资源（图标、截图）
- 分类和语言
- 定价（选择免费）

### 4. 填写隐私权规范

**关键步骤！必须填写权限理由**

- 单一用途说明
- 权限理由（复制第 2 步的文本）
- 隐私政策
- 认证披露

### 5. 设置分发范围

- **可见性：** 公开
- **地区：** 所有地区（或选择特定地区）

### 6. 提交审核

- 检查所有信息
- 点击「提交审核」
- 等待审核结果

---

## ⏱️ 审核时间

- **一般情况：** 1-3 个工作日
- **首次发布：** 可能需要 5-7 个工作日
- **包含主机权限：** 可能需要深入审核

---

## 🔍 常见拒绝原因

### 1. 权限理由不充分

**解决方案：**
- 参考第 2 步的详细说明
- 强调功能必要性
- 说明隐私保护措施

### 2. 截图不清晰

**解决方案：**
- 使用 1280×800 高清截图
- 展示核心功能
- 添加文字说明（可选）

### 3. 描述不清楚

**解决方案：**
- 清楚说明扩展程序的用途
- 列举主要功能
- 提供使用步骤

### 4. 单一用途不明确

**解决方案：**
- 用一句话概括核心功能
- 不要包含多个不相关的功能

---

## 📞 如果被拒绝

### 步骤 1：查看拒绝原因
