# 🔧 问题排查指南

## 问题：点击没反应

### 步骤 1：确认插件已正确安装

1. 打开 Chrome，访问：`chrome://extensions/`
2. 查找「网页表格识别与导出器」
3. 确认状态为「已启用」
4. **重要**：检查是否有错误提示（红色文字）

### 步骤 2：检查图标文件是否存在

插件需要三个图标文件才能正常运行：

```bash
# 在项目目录下检查
ls -la icon*.png
```

如果没有图标文件，请：
1. 在浏览器打开 `icon-generator.html`
2. 下载三个图标文件
3. 放入项目根目录
4. 在 `chrome://extensions/` 中点击刷新按钮 🔄

### 步骤 3：检查页面表格类型

**重要发现**：从您的截图看，这个表格可能不是标准的 HTML `<table>` 元素！

#### 如何验证：

1. 在页面上右键 → 选择「检查」
2. 点击左上角的元素选择器（或按 Ctrl/Cmd + Shift + C）
3. 点击表格区域
4. 查看 Elements 面板中显示的 HTML 结构

**如果是 `<table>` 标签** → 插件应该能识别
**如果是 `<div>` 或其他标签** → 插件无法识别（需要修改代码）

### 步骤 4：查看浏览器控制台错误

1. 右键点击页面 → 选择「检查」
2. 切换到「Console」标签
3. 刷新页面
4. 查看是否有红色错误信息

常见错误：
- `Manifest file is missing or unreadable` → 图标文件缺失
- `Content Security Policy` → 页面安全策略限制
- JavaScript 错误 → 代码问题

### 步骤 5：使用测试页面验证

先在测试页面验证插件是否正常工作：

```bash
# 在浏览器中打开
test-page.html
```

如果测试页面正常，说明插件没问题，是目标网页的表格类型不兼容。

## 🚨 最可能的原因

从截图判断，这个表格很可能是：
- 数据库管理工具的查询结果界面
- 使用 `<div>` + CSS 渲染的表格
- 而不是标准的 HTML `<table>` 元素

**当前插件只支持 HTML `<table>` 元素！**
