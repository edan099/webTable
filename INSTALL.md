# 🚀 安装指南

## 第一步：准备图标文件

由于 Chrome 插件必须包含图标文件，您需要先生成图标：

### 方法一：使用图标生成器（推荐）

1. 在浏览器中打开 `icon-generator.html` 文件
2. 点击三个下载按钮，分别下载：
   - `icon16.png` (16×16)
   - `icon48.png` (48×48)
   - `icon128.png` (128×128)
3. 将下载的三个图标文件放入项目根目录（与 manifest.json 同级）

### 方法二：使用自定义图标

如果您有自己的图标，请确保：
- 准备三个尺寸的 PNG 图片：16×16、48×48、128×128
- 命名为：`icon16.png`、`icon48.png`、`icon128.png`
- 放入项目根目录

### 方法三：临时跳过图标（开发测试）

如果只是临时测试，可以从网上下载任意图标文件，或暂时注释掉 `manifest.json` 中的 icons 配置：

```json
{
  ...
  // "icons": {
  //   "16": "icon16.png",
  //   "48": "icon48.png",
  //   "128": "icon128.png"
  // }
}
```

## 第二步：加载插件到 Chrome

### 1. 打开扩展程序页面

在 Chrome 浏览器地址栏输入：
```
chrome://extensions/
```

### 2. 启用开发者模式

在扩展程序页面右上角，打开「**开发者模式**」开关

### 3. 加载插件

1. 点击左上角的「**加载已解压的扩展程序**」按钮
2. 选择本项目文件夹（webTable 文件夹）
3. 点击「选择」

### 4. 验证安装

如果一切顺利，您将看到：
- ✅ 插件名称：**网页表格识别与导出器**
- ✅ 版本：**1.0.0**
- ✅ 状态：**已启用**

## 第三步：测试插件功能

### 使用测试页面

1. 在浏览器中打开 `test-page.html` 文件
2. 将鼠标移动到任意表格上
3. 右上角会出现「📊 提取表格」按钮
4. 点击按钮，测试各种导出功能

### 在真实网页上测试

访问任何包含表格的网站，例如：
- Wikipedia 维基百科
- GitHub 仓库文件列表
- 数据统计网站
- 新闻网站的数据表格

## 常见问题

### ❓ 插件加载失败

**可能原因 1：缺少图标文件**
- 检查项目根目录是否存在三个图标文件
- 或临时注释掉 manifest.json 中的 icons 配置

**可能原因 2：manifest.json 格式错误**
- 检查 JSON 文件格式是否正确
- 不要有多余的逗号或注释

**可能原因 3：文件编码问题**
- 确保所有文件使用 UTF-8 编码

### ❓ 悬浮按钮不显示

**检查项：**
1. 确认页面中确实存在 `<table>` 元素
2. 打开开发者工具（F12），查看 Console 是否有错误
3. 刷新页面重试
4. 检查插件是否已启用

### ❓ 复制功能不工作

**可能原因：**
- 浏览器权限限制
- 尝试先点击页面任意位置，再使用复制功能
- 某些网站可能有安全策略限制

### ❓ 样式显示异常

**解决方案：**
1. 刷新页面
2. 检查是否与页面本身的 CSS 冲突
3. 尝试在隐私模式下测试

## 打包为 CRX 文件（可选）

如果您想将插件打包为 `.crx` 文件以便分发：

### 方法一：Chrome 内置打包

1. 访问 `chrome://extensions/`
2. 点击「打包扩展程序」
3. 选择项目文件夹
4. 点击「打包扩展程序」按钮
5. 生成 `.crx` 文件和 `.pem` 密钥文件

### 方法二：命令行打包

```bash
# 需要安装 Chrome 浏览器
# 在项目目录下执行
chrome --pack-extension=/path/to/webTable
```

## 卸载插件

如需卸载插件：

1. 访问 `chrome://extensions/`
2. 找到「网页表格识别与导出器」
3. 点击「移除」按钮

## 更新插件

修改代码后，需要重新加载：

1. 访问 `chrome://extensions/`
2. 找到插件，点击 🔄 刷新按钮
3. 或者点击「重新加载」链接
