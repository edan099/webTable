# 🧪 测试指南：SQL 自定义表名功能

## 快速测试步骤

### 第 1 步：刷新插件 🔄

```bash
1. 访问：chrome://extensions/
2. 找到「网页表格识别与导出器」
3. 点击刷新按钮 🔄
4. 确认版本号为 v1.2.0
```

---

### 第 2 步：打开测试页面

已为您打开 `test-page.html`

---

### 第 3 步：测试自定义表名

#### 测试 1：默认表名

1. 鼠标移到任意表格上
2. 点击「📊 提取表格」
3. 点击「🟣 导出 SQL」标签
4. **观察：黄色配置区出现**
5. **观察：默认表名为 `table_data`**

**预期输出：**
```sql
-- INSERT 语句

INSERT INTO table_data (姓名, 职位, 部门, 入职日期, 薪资) VALUES ('张三', '前端工程师', '技术部', '2023-01-15', 15000);
```

---

#### 测试 2：修改为英文表名

1. 在输入框中输入：`employees`
2. 按 **Enter** 键

**预期输出：**
```sql
INSERT INTO employees (姓名, 职位, 部门, 入职日期, 薪资) VALUES ('张三', '前端工程师', '技术部', '2023-01-15', 15000);
```

✅ 表名已更改为 `employees`

---

#### 测试 3：修改为中文表名

1. 在输入框中输入：`员工表`
2. 点击 **🔄** 刷新按钮

**预期输出：**
```sql
INSERT INTO 员工表 (姓名, 职位, 部门, 入职日期, 薪资) VALUES ('张三', '前端工程师', '技术部', '2023-01-15', 15000);
```

✅ 支持中文表名

---

#### 测试 4：特殊字符清理

1. 在输入框中输入：`user-data@2024`
2. 按 Enter

**预期输出：**
```sql
INSERT INTO user_data_2024 (姓名, 职位, 部门, 入职日期, 薪资) VALUES ...
```

✅ 特殊字符自动转换为下划线

---

#### 测试 5：快速切换表名

1. 输入 `users` → Enter → 复制 SQL
2. 输入 `user_profiles` → Enter → 复制 SQL
3. 输入 `user_data` → Enter → 复制 SQL

✅ 可以快速生成多个不同表名的 SQL

---

### 第 4 步：测试 UI 交互

#### 测试配置区显示/隐藏

1. **点击「🟢 导出 JSON」**
   - 配置区应该隐藏

2. **点击「🔵 导出 CSV」**
   - 配置区应该隐藏

3. **点击「🟣 导出 SQL」**
   - 配置区应该显示

✅ 配置区仅在 SQL 标签下显示

---

#### 测试刷新按钮动画

1. 鼠标悬停在 **🔄** 按钮上
2. **观察：按钮应该旋转 90 度**

✅ 动画效果正常

---

#### 测试输入框聚焦效果

1. 点击输入框
2. **观察：边框变为深金色**
3. **观察：有淡淡的阴影效果**

✅ 聚焦效果正常

---

### 第 5 步：完整功能测试

#### 完整流程

1. **选择表格** → 悬停 → 点击提取
2. **切换到 SQL** → 配置区出现
3. **输入表名** → `my_table`
4. **按 Enter** → SQL 重新生成
5. **点击复制** → 复制到剪贴板
6. **粘贴验证** → Ctrl/Cmd + V

**预期结果：**
```sql
INSERT INTO my_table (...) VALUES (...);
```

✅ 完整流程正常
