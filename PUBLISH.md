## 📦 方式一：打包为 CRX 文件（本地分发）

### 适用场景
- 企业内部使用
- 朋友间分享
- 测试和开发

### 打包步骤

#### 方法 1：Chrome 内置打包（推荐）

1. **打开 Chrome 扩展页面**
   ```
   chrome://extensions/
   ```

2. **开启开发者模式**
   - 点击右上角「开发者模式」开关

3. **点击「打包扩展程序」**
   - 在页面顶部找到「打包扩展程序」按钮

4. **选择扩展程序根目录**
   ```
   /Users/didi/IdeaProjects/webTable
   ```
   - **重要**：不要选择父目录，要选择包含 manifest.json 的文件夹

5. **首次打包**
   - 私有密钥文件：留空（会自动生成）
   - 点击「打包扩展程序」

6. **生成的文件**
   ```
   webTable.crx         # 打包好的插件文件
   webTable.pem         # 私钥文件（重要！妥善保管）
   ```

7. **二次更新打包**
   - 扩展程序根目录：`/Users/didi/IdeaProjects/webTable`
   - 私有密钥文件：选择之前生成的 `.pem` 文件
   - 点击「打包扩展程序」

#### 方法 2：命令行打包

```bash
# 需要 Chrome 浏览器
# Mac
/Applications/Google\ Chrome.app/Contents/MacOS/Google\ Chrome \
  --pack-extension=/Users/didi/IdeaProjects/webTable

# Windows
"C:\Program Files\Google\Chrome\Application\chrome.exe" \
  --pack-extension=C:\path\to\webTable
```

### 安装 CRX 文件

#### 方法 1：拖拽安装
1. 打开 `chrome://extensions/`
2. 开启「开发者模式」
3. 将 `.crx` 文件拖拽到页面中
4. 点击「添加扩展程序」

#### 方法 2：加载已解压
1. 打开 `chrome://extensions/`
2. 开启「开发者模式」
3. 点击「加载已解压的扩展程序」
4. 选择项目文件夹

### ⚠️ CRX 文件的限制

从 Chrome 73 开始，直接安装 `.crx` 文件受到限制：
- 只能通过 Chrome Web Store 安装
- 或使用「加载已解压的扩展程序」（开发者模式）
- 企业可通过策略部署

**建议：优先考虑发布到 Chrome Web Store**

### 发布步骤

#### 第 1 步：注册开发者账号

1. **访问 Chrome Web Store Developer Dashboard**
   ```
   https://chrome.google.com/webstore/devconsole
   ```

2. **注册开发者**
   - 使用 Google 账号登录
   - 支付一次性注册费用：**$5 USD**
   - 填写开发者信息

3. **等待审核**
   - 通常 1-3 个工作日

#### 第 2 步：上传扩展程序

1. **进入开发者控制台**
   - 点击「新增项」按钮

2. **上传 ZIP 文件**
   - 上传 `webTable-v1.1.1.zip`
   - 系统自动验证

3. **填写商品详情**
   - 商品名称
   - 简短描述
   - 详细描述
   - 类别选择
   - 语言（中文简体）

4. **上传宣传素材**
   - 小图标（128×128）
   - 宣传图（440×280）
   - 屏幕截图（1-5 张）

5. **设置权限说明**
   ```
   activeTab - 访问当前标签页，用于提取表格数据
   ```

6. **选择发布区域**
   - 建议：全球发布
   - 或指定国家/地区

7. **设置价格**
   - 免费

8. **提交审核**
   - 点击「发布」按钮
   - 等待 Google 审核

#### 第 3 步：等待审核

**审核时间：**
- 通常：1-3 个工作日
- 高峰期：5-7 个工作日
- 首次发布可能更长

**审核内容：**
- 代码安全性
- 权限合理性
- 隐私政策（如需）
- 内容合规性

**可能被拒原因：**
- 权限过度申请
- 代码存在安全问题
- 违反内容政策
- 缺少隐私政策（如收集数据）

#### 第 4 步：发布成功

审核通过后：
- 插件在 Chrome Web Store 上线
- 用户可搜索并安装
- 获得唯一的插件 ID
- 可以看到安装量和评价
